ブランチとは？
・ブランチとは、履歴の流れを分岐して記録していくためのものです。分岐したブランチは他のブランチの影響を受けないため、同じリポジトリ中で複数の変更を同時に進めていくことができます。
~さる先生のGit入門より抜粋~
https://backlog.com/ja/git-tutorial/stepup/01/

1:リモート上の差分を取得する
git pull
もしくは
git fetch
git merge origin/ブランチ名

2:ブランチ作成
git branch 新しいブランチ名

3:ブランチ名を確認するためのコマンド
git branch

4:ブランチ移動
git checkout 移動したいブランチ名

5:変更されたファイルの確認
git status

6:変更を登録する（コミットするために必要な工程）
git add ファイル名

7:登録したファイルをコミットする
git commit -m 'コミットメッセージ'

8:リモートとローカルのファイルの差分を抽出する
git diff ファイル名

9:コミット履歴を確認する
git log

10:リモートにプッシュ
git push origin HEAD

11:ブランチ削除
git branch -D
